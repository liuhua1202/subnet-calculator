# Subnet Calculator v1.0.5 Release Notes

This release is a **code-review hardening batch**: 27 issues surfaced during a full-project audit, 19 real bugs + 5 engineering improvements shipped together. All user-visible changes are backward-compatible.

## 🐛 Fixes

- **IP type detection extended**: now recognises RFC 5737 documentation ranges (`192.0.2.0/24`, `198.51.100.0/24`, `203.0.113.0/24`) and RFC 6598 CGNAT (`100.64.0.0/10`) — no longer mis-classified as public
- **IP type now judged by network address**: `192.168.1.5 /16` correctly shows "private" (previously the host IP `192.168.1.5` was classified as public C-class)
- **/31 and /32 hints**: these edge cases now show explanatory sub-text (RFC 3021 point-to-point / single-host)
- **Toast early-dismiss bug**: `setTimeout(1800)` was out of sync with the 2100ms animation, so the toast vanished mid fadeOut. Replaced with an `animationend` listener
- **CI release body root cause fixed**: the `release` job was missing `actions/checkout`, so `body_path` could never resolve the notes file. Now the release job checks out the source
- **a11y reference**: `<input aria-describedby="ipInput-hint">` now points to a real `#ipInput-hint` element (previously a dangling reference)
- **Electron blank-on-failure**: `did-fail-load` now pops a `dialog.showErrorBox` with the error code and description
- **debounce + change race**: the mask `<select>` change handler now clears the input debounce timer to avoid a duplicate calculation 400ms later
- **Default IP placeholder removed**: input no longer ships with a hardcoded `value="192.168.1.1"` (was misleading to first-time users)

## ✨ Improvements

- **Frontend split into modules**: `www/index.html` (UI / styles) + new `www/subnet.js` (pure-function library) + new `www/app.js` (DOM / event handlers) — paved the way for unit tests
- **CSP tightened**: all `<script>` are now external, `script-src 'unsafe-inline'` removed
- **Mobile theme-color**: `<meta name="theme-color" content="#0078d4">` so Android status bar matches the brand
- **IP type label renamed "IP类型" → "网络类型"** for semantic accuracy (classification is by network address, not host)

## 🛠️ Engineering

- **Vitest unit tests**: 55 cases cover all IP / mask / CIDR conversions and IP type recognition (`npm test`)
- **ESLint 9**: flat-config style with no-undef / eqeqeq / prefer-const and friends (`npm run lint`)
- **`engines` field**: declares `node >= 20, npm >= 9` to keep low-version users from getting stuck
- **7za postinstall actually works now**: `package.json` adds a `postinstall` hook that runs `build/7za-wrapper/build.js`, which auto-compiles the C# shim on Windows. The README previously claimed this happened but `build.js` did not exist
- **Program.cs compatible with older csc.exe**: uses `Arguments` string + manual quoting instead of `ProcessStartInfo.ArgumentList` (the latter needs .NET Core 2.1+)
- **Electron build excludes test code**: `build.files` adds `!**/tests/**` and `!**/eslint.config.*`

## 📚 Docs

- Removed the false "iOS (Capacitor, source only)" claim
- 7za section: now describes the real workflow (`postinstall` → `build.js` → compile C# shim)
- Fixed `8.8.8.8 /8` example: broadcast `15.255.255.255` → `8.255.255.255` (the README was wrong; the code has always been right)
